## 一、总体说明

> 说明本次优化的背景、触发原因、整体目标和原则。

本次 UI 风格优化的背景是：[项目] 系统历史页面由多名开发人员分批次迭代完成，各模块在控件规格、表格渲染、操作按钮形式、颜色使用等方面存在明显不一致，导致用户使用时体感割裂、视觉混乱。

## 二、UI 设计架构

### 2.1 设计定位

本项目采用 **Stripe 精致 × Linear 密度** 融合风格：高密度（`size=small`、`margin-bottom:8px`）、精致感（圆角 6–12px、微阴影、hover/focus 过渡）、一致性（所有颜色来自 CSS Token，主题色只需改一处全局生效）。

### 2.2 Token 体系（单一来源）

```
Token 入口：[Token 文件路径]
主色:     var(--el-color-primary)    = [颜色值]
成功:     var(--el-color-success)    = [颜色值]
警告:     var(--el-color-warning)    = [颜色值]
危险:     var(--el-color-danger)     = [颜色值]
文字一级: var(--el-text-color-primary)
文字二级: var(--el-text-color-regular)
文字三级: var(--el-text-color-secondary)
边框:     var(--el-border-color)
背景浅:   var(--el-fill-color-lighter)
背景卡:   var(--el-bg-color)
```

> ⚠️ 禁止在业务文件中硬编码任何 HEX/RGB 颜色

### 2.3 样式分层策略

```
wl-skills-ui presets/[preset].scss    ← 第1层：Token 变量 + L1 element + L2 vendors（自动覆盖）
src/assets/style/element.scss         ← 第2层：EP 控件局部补丁
src/assets/style/main.scss            ← 第3层：全局共享组件样式
views/[module]/style/index.scss       ← 第4层：模块级布局补丁（仅限布局，禁止硬编码颜色）
业务组件 <style scoped>               ← 第5层：组件内布局，禁止硬编码颜色
```

### 2.4 覆盖优先级规则

| 场景 | 写法 |
|------|------|
| jh-ui 组件（用了 `html` 选择器） | `html .el-xxx { ... !important }` |
| Element Plus scoped 样式 | 全局文件用非 scoped 选择器 + `!important` |
| 业务组件内需穿透子组件 | `:deep(.el-xxx)` |
| 远程 Module Federation 组件 | CSS 变量注入（`--el-color-primary` 等） |
| jh-dialog footer 按钮 | `.el-dialog__footer .el-button`（无 wrapper class） |

### 5.2 操作列按钮系统 — 统一图标化

> 对应规范：R004 / R013

| 优化项 | 改造前 | 改造后 |
|--------|--------|--------|
| 操作按钮形式 | 文字按钮，占 120~200px 列宽 | 26×26px 纯图标按钮，hover 变语义色 |
| 按钮间距 | 不统一 | `renderOps` 自动控制 gap:4px |

**防漏规则**：字段名含 `Code / No / no / code` 或 label 含"编号"的列，必须在 `columnsDef` 中显式调用 `renderBadge`；`queryDef`（搜索表单）中不需要。

## 十、技术约束备忘

| 约束 | 说明 |
|------|------|
| `!important` | jh-ui 用 `html` 选择器（特异性极高），覆盖必须加 |
| scoped 无效 | 全局覆盖写全局文件，业务组件不写全局 CSS |
| Module Federation | 远程组件只能通过 CSS 变量影响，不能直接改源码 |
| jh-dialog footer 按钮 | 按钮直接在 `#footer` slot，无 `.dialog-footer` wrapper，需用 `.el-dialog__footer .el-button` |
| renderBadge 规则 | 字段名含 Code/No/编号 → 必须在 `columnsDef` 中显式调用 `renderBadge(row.xxx)` |
| queryDef 中不适用 | `queryDef` 是搜索表单，不是表格列，不需要 `defaultSlot` |
